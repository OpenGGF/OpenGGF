35 Unsupported Rewind Fields Orchestration Report
=================================================

Requirements
------------

Goal: remove the 35 unsupported rewind-field inventory entries without adding
per-object annotation noise. Non-goals: full encounter/torture validation and
bespoke child-graph rewind semantics. Constraints: keep trace data diagnostic
only, prefer shared schema codecs and central policy, preserve structural
runtime references instead of serializing services/events/suppliers.

Acceptance criteria:
- RewindFieldInventoryTool reports no unsupported fields.
- Main code compiles.
- Repeated field shapes are handled by shared codecs/policy.
- Player references in compact object state use RewindIdentityTable context.

Exploration Synthesis
---------------------

Three bounded explorations grouped the fields into:
- Structural runtime handles: services, sensors, events, suppliers, child graph,
  Caterkiller parent link.
- Plain helper state: boss state/flasher/sequencer, vine segments, AIZ swing
  motion, result elements, palette colors.
- Player-keyed collections: launcher state, CNZ/HCZ rider state, MGZ/pachinko
  player maps, primitive array values.

Recommendation: add central transient policies for structural dependencies,
extend compact codecs for plain holders/arrays/Palette.Color/collection values,
and pass identity context through object compact capture.

Architecture Decision
---------------------

Ownership remains in `game.rewind.schema` and existing object rewind plumbing.
No new per-zone registry or per-object snapshot class was introduced. Runtime
references are policy-excluded; mutable gameplay helpers are captured through
schema codecs; player references are encoded via `RewindIdentityTable`.

Feature Design
--------------

`RewindCodecs` now supports:
- `Palette.Color` values and arrays.
- Arrays of supported plain state holders.
- Constructible plain state holders as collection/map values.
- Player-reference fields inside nested state holders.
- Primitive-array collection values.

`RewindPolicyRegistry` now covers structural services, event handles, supplier
closures, sensors, player power-up handles, zero-state handlers, boss child
graphs, and Caterkiller parent links.

Implementation Plan
-------------------

1. Add failing schema/helper tests for policy and codec gaps.
2. Extend codecs and central policies.
3. Thread `RewindCaptureContext` through default object compact capture.
4. Run compile and inventory verification.

-----------------

`testCompile` state currently reports unrelated missing-symbol errors in many
test sources, so full targeted JUnit reruns were blocked after the code changes.
The first schema-test run proved the new tests failed for the intended missing
support before implementation.
