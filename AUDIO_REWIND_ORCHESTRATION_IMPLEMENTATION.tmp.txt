Audio Rewind Orchestration Implementation Report
================================================

Requirements
------------

Goal: advance `AUDIO_REWIND_BLUEPRINT.tmp.txt` from capture-only Tier 2
logical snapshots toward restorable audio keyframes and live rewind logical
state rebuild.

Acceptance for this implementation slice:
- AudioManager logical snapshots can be restored.
- Backend logical snapshots have an explicit restore contract.
- AudioKeyframeStore restores the nearest audio keyframe before replaying
  later timeline commands.
- RewindController restores audio logical state during live seek/backstep
  without emitting presentation backend calls.
- SMPS driver snapshots expose compact source descriptors and preserve
  fallback voice links across in-process restore.
- SMPS driver snapshots can be restored through a caller-provided dependency
  resolver without relying on captured live dependency identities.
- SMPS source descriptors carry route identity for later loader-backed restore.
- Backend logical snapshots carry optional SMPS driver snapshots, and
  AudioManager supplies a route-aware resolver to backend restore.
- Restored LWJGL fade-in snapshots rebind fade-complete callbacks needed to
  clear SFX blocking.
- PSG, YM2612, DAC playback, resampler, mixed synth, and SMPS driver snapshots
  can restore future sample output bit-exactly for representative coverage.
- `LWJGLAudioBackend.restoreLogicalSnapshot(...)` can restore a captured SMPS
  driver snapshot through the backend path while preserving bit-exact future
  mixed samples.
- Existing Tier 0 suppression and Tier 1 command-timeline tests remain green.

Non-goals for this slice:
- No audible rewind presentation or release crossfade work.
- No cycle-level command/chip-write timing model.

Exploration Synthesis
---------------------

Explorer agreement:
- `AudioKeyframeStore` replayed commands after the nearest keyframe but did
  not restore the captured `AudioLogicalSnapshot`.
- `AudioBackend` had capture-only logical snapshot support.
- `LWJGLAudioBackend.captureLogicalSnapshot()` exposed descriptor/state fields
  but had no restore path.
- SMPS driver/sequencer snapshots exist and round-trip unit state, but still
  use live in-process references and are not durable descriptor snapshots.

Architecture Decision
---------------------

Add a narrow restore seam matching the existing capture seam:
- `AudioCommandTimeline.restoreCursor(frame, nextOrder)` restores timeline
  cursor metadata without truncating recorded history.
- `AudioManager.restoreLogicalSnapshot(snapshot)` restores manager-owned
  intent state and delegates backend logical state restore.
- `AudioBackend.restoreLogicalSnapshot(snapshot)` is a default no-op for
  non-participating backends.
- `LWJGLAudioBackend.restoreLogicalSnapshot(snapshot)` restores descriptor
  and logical flags while clearing presentation-owned stream references.

This keeps OpenAL presentation state out of authoritative rewind state.

Feature Design
--------------

`AudioKeyframeStore.replayTo(...)` now:
1. Finds the nearest keyframe.
2. Restores its `AudioLogicalSnapshot`.
3. Replays timeline entries after the captured command-entry count up to the
   target frame.

`AudioKeyframeStore.replayToLogicalState(...)` now uses a separate logical
command replay path. `RewindController` uses that path after restoring gameplay
state and before resuming normal audio frame recording, so internal
seek/backstep replay rebuilds ring alternation and descriptor-level backend
intent without presentation calls.

Future audio keyframes are pruned with future gameplay keyframes and future
audio commands after a rewind branch.

`SmpsSourceDescriptor` now records stable source metadata for SMPS sequencers.
`SmpsDriverSnapshot.SequencerEntry` carries the sequencer source and optional
fallback voice source, and `SmpsDriver.restoreSnapshot(...)` rebinds fallback
voice data between restored sequencers by descriptor match.

`SmpsDriver.restoreSnapshot(snapshot, resolver)` now pre-resolves and validates
all sequencer dependencies before taking the driver lock and mutating state.
This keeps failure atomic when a resolver cannot provide a required source or
provides data that does not match the snapshot descriptor.

`SmpsSourceDescriptor` now records route identity plus validation metadata.
`SmpsSequencer` owns a source descriptor, `SmpsDriver` captures that descriptor
and uses same-driver descriptor lookup for fallback voice sources, and
`LWJGLAudioBackend` assigns route-aware descriptors when it creates music and
SFX sequencers.

`AudioBackendLogicalSnapshot` now carries optional music-driver and
standalone-SFX-driver snapshots. `AudioManager.restoreLogicalSnapshot(...)`
passes a route-aware `SmpsDriverSnapshot.DependencyResolver` to the backend, and
`LWJGLAudioBackend` uses it to rebuild captured SMPS drivers while still
flushing OpenAL presentation state.

When restored backend state is SFX-blocked and the restored primary music
sequencer is in active fade-in, `LWJGLAudioBackend` rebinds the fade-complete
callback so the restored sequencer can clear `sfxBlocked` when fade-in
finishes.

Tier 3 chip/sample state now has a manual snapshot seam:
- `PsgChipGPGX` snapshots include PSG counters, polarity/output state, noise
  LFSR state, mute/delta state, and `BlipDeltaBuffer` history.
- `Ym2612Chip` snapshots include FM channel/operator state, timers, LFO/EG
  counters, DAC playback/latched/high-pass state, and `BlipResampler` history.
- `VirtualSynthesizer` snapshots combine YM and PSG state.
- `SmpsDriverSnapshot` carries the synth snapshot and driver restore applies it
  after rebuilding logical sequencers, avoiding the old chip-silencing path for
  chip-authoritative snapshots.

End-to-End Review
-----------------

No blocker in the completed slice.

Residual risks and deferrals:
- SMPS snapshots still carry live references for the legacy compatibility
  restore path, although resolver-backed restore no longer depends on them.
- Fade-complete callbacks are structural backend behavior and are rebound by
  LWJGL restore, not captured by generic driver snapshot restore.
- Cycle-level command/chip-write timing and audible reverse PCM presentation
  remain later-tier work.
