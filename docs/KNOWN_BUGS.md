# Known Bugs and Unfinished Work (Engine-Wide)

This document tracks **bugs**, incomplete implementations, and known parity gaps that we intend to fix but haven't addressed yet. Entries here are *not* intentional — they're acknowledged problems with a plan (or hope) of eventual resolution.

For **intentional** deviations from the original ROMs (architectural choices, feature extensions, deliberate bug-fixes of ROM data), see [KNOWN_DISCREPANCIES.md](KNOWN_DISCREPANCIES.md).

Entries should include:
- **Location** — the file(s) where the bug lives, if known
- **Symptom** — what goes wrong and where you can observe it (test name, trace frame, manual repro)
- **Suspected cause** — best current theory, with ROM/disasm references when relevant
- **Removal condition** — what needs to be true for this entry to be deleted

## Trace Replay Recorder Coverage Follow-Up

**Location:** `src/test/java/com/openggf/tests/trace/*`, `tools/bizhawk/*`

### Symptom

BK2-derived fixture coverage is still incomplete across the full replay suite. Sonic 1 and Sonic 2 ship with established BK2-derived trace directories, and S3K now has a current AIZ end-to-end fixture generated by the S3K recorder (`lua_script_version=6.25-s3k`). Remaining S3K routes such as CNZ/MGZ should be regenerated with the current recorder when they next need parity work, rather than adding legacy fixture compatibility.

### Current State

The shared replay harness understands execution counters and uses `gameplay_frame_counter` plus `vblank_counter` when those columns are present. The Sonic 1, Sonic 2, and Sonic 3&K BizHawk recorders all emit current counter-bearing schemas, and committed real fixtures cover all three games. The release guard now rejects accepted `isLegacy...Trace` replay predicates, so old route fixtures should be regenerated instead of being special-cased.

`TraceData` now logs a one-shot notice when a pre-v3 trace directory is loaded so the fallback is visible during test runs.

### Removal Condition

Remove this entry once the remaining committed S3K route fixtures that predate the current recorder are regenerated and the generic missing-counter fallback in `TraceExecutionModel` / `TraceData` is no longer needed by any fixture or synthetic parser test.
