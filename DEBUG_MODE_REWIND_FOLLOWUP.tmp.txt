Debug Mode Rewind Follow-Up
===========================

Purpose
-------

This note expands the post-audio follow-up task from
REWIND_SUPPORT_BLUEPRINT.tmp.txt: support rewinding across level debug-mode
entry, debug movement, and debug-mode exit.

Current Findings
----------------

Level debug mode is gameplay-affecting, not cosmetic. Objects and zone features
branch on `player.isDebugMode()`, so replaying across debug-mode boundaries
must restore both the player flag and the input events that toggle it.

Relevant current paths:
- `SpriteManager.update(...)` reads `DEBUG_MODE_KEY` and toggles each playable.
- `PlayableSpriteMovement.handleDebugMovement(...)` directly mutates player
  position while debug mode is active.
- `AbstractPlayableSprite.captureRewindState(...)` captures position and many
  physics fields, but currently does not capture `debugMode`.
- `LiveRewindInputSource` records ordinary movement/jump/P2 rows only.
- `RewindFrameInputHandler` reconstructs only those recorded ordinary controls.

Required Work
-------------

1. Capture and restore `AbstractPlayableSprite.debugMode`.
2. Extend live rewind input rows to record debug-mode toggle edges.
3. Record debug movement modifiers used by debug movement, such as shift/ctrl
   speed changes, so replayed debug movement matches live movement.
4. Record debug test/option key edges if they affect gameplay state during
   level debug mode.
5. Extend `RewindFrameInputHandler` to reconstruct these keys during internal
   rewind replay.
6. Add tests that enter debug mode, move in debug mode, exit debug mode, rewind
   across each boundary, replay, and compare player state.

Scope Boundary
--------------

The first implementation should cover level debug mode movement and enter/exit
state only. Debug object placement, debug object cycling, and any spawned debug
objects may need a separate command-log design if they mutate object layout or
runtime object graphs.

Acceptance Criteria
-------------------

- Rewind restore inside debug mode returns the player to debug mode.
- Rewind restore before debug entry returns the player to normal gameplay mode.
- Rewind+replay across debug entry, debug movement, and debug exit produces the
  same player position and debug flag as uninterrupted play.
- Normal non-debug live rewind behavior remains unchanged.
